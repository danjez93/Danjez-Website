How I Work

A process built for complexity, not comfort.

Every project moves through five phases — not linearly, but deliberately. I document decisions as I make them so teams can build on the work, not reconstruct it.

01
Discover

Stakeholder interviews, competitive analysis, heuristic evaluation, analytics review.

02
Define

Problem framing, user personas, journey maps, design principles for the project.

03
Design

Wireframes, high-fidelity UI, prototype, component specifications.

04
Validate

Usability testing, A/B experiments, accessibility audit, dev collaboration.

05
Measure

Success metrics tracking, iteration roadmap, design system updates.
