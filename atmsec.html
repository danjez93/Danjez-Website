ATM Sec — Monitoring at Scale

A B2B operations platform for managing and monitoring ATM networks across multiple clients — designed for operators who need to act fast on incomplete information.

Overview

When hundreds of ATMs break, someone needs to see everything at once

ATM Sec is a monitoring platform used by B2B operators to manage ATM networks on behalf of banks and financial institutions. Operators can have 200–800 machines in their portfolio, spread across multiple client accounts, cities, and failure types.

The existing interface was built by engineers for engineers — tables of raw data, no visual hierarchy, no prioritization. My role was to redesign the core dashboard and alert management system into something operators could actually act on under pressure.

Core Problem

"Operators were missing critical alerts because everything looked equally urgent. The interface created noise, not signal."

01 — Benchmarking

Analyzing the existing system and the competition

I started by auditing the existing interface and benchmarking 6 operations dashboards in adjacent industries: network monitoring (Datadog, PagerDuty), field service management, and ATM-specific tools from NCR and Diebold.

Existing interface audit — problems mapped

Benchmarking — Datadog, PagerDuty, NCR

Pattern library from competitive analysis

Key patterns I extracted: severity-based color coding (not just red/green), geographic clustering for field dispatch, and progressive disclosure for alert details — reveal context on demand, not all at once.

02 — Information Architecture

What operators actually need to see first

Through 5 contextual observation sessions with ATM operators, I identified three decision types that drive the operator's day: triage (which ATM needs attention right now), dispatch (who goes where), and reporting (what happened this week).

I restructured the IA around these three modes rather than the data model the engineers had built around. The navigation became mode-based, not table-based.

Information architecture — before / after comparison

Restructured navigation: 3 modes replacing 11 flat tabs

03 — UI Design

Dashboard, alert management, and map view

I designed three core views. The main dashboard gives operators a status overview at a glance — machine count by severity, geographic distribution, and the top 5 alerts requiring action. The alert view allows filtering, sorting, and bulk actions. The map view shows machine locations with color-coded status overlays.
